# 运维文档库

## 简介

本仓库用于存储和管理运维相关的技术文档，实现文档的版本控制、协作编辑和历史追溯。

## 目录结构

```
.
├── docs/                    # 运维文档目录
│   ├── 数据库/              # 数据库运维文档（MySQL、Redis、MongoDB等）
│   ├── 中间件/              # 中间件运维文档（Nginx、Tomcat、Kafka等）
│   ├── 系统/                # 操作系统运维文档（Linux、Windows等）
│   ├── 网络/                # 网络配置与运维文档
│   ├── 安全/                # 安全相关文档
│   └── 监控/                # 监控系统文档（Zabbix、Prometheus等）
├── templates/               # 文档模板目录
├── scripts/                 # 辅助脚本目录
├── .gitignore               # Git忽略配置
├── README.md                # 本说明文件
└── CONTRIBUTING.md          # 文档规范说明
```

## 文档规范

请参阅 [CONTRIBUTING.md](CONTRIBUTING.md) 了解文档编写规范和提交要求。

## 使用说明

### 克隆仓库

```bash
git clone <仓库地址>
```

### 添加新文档

1. 在对应目录下创建Markdown文件
2. 按照文档规范编写内容
3. 提交更改

### 更新文档

1. 编辑现有文档
2. 在文档底部更新修订记录
3. 提交更改并注明修改内容

## 文档命名规范

- 使用中文命名，简洁明了
- 格式：`组件名称运维手册.md` 或 `功能描述.md`
- 示例：`Redis运维手册.md`、`MySQL主从复制配置.md`

## 版本控制

- 每次更新文档需在文档底部记录修订历史
- 提交信息格式：`[类型] 简要描述`
  - 类型：新增、更新、修复、删除
  - 示例：`[更新] Redis运维手册-补充集群配置章节`

## 联系方式

如有问题或建议，请联系运维团队。
