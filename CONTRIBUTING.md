# 文档规范说明

## 文档编写规范

### 1. 文档格式

所有运维文档统一使用 Markdown 格式编写，文件编码为 UTF-8。

### 2. 文档结构

一份完整的运维文档应包含以下部分：

```markdown
# 文档标题

## 一、概述
简要描述文档内容和适用场景。

## 二、环境要求
列出所需的软硬件环境、版本要求等。

## 三、安装部署
详细的安装步骤和配置说明。

## 四、配置说明
关键配置项的解释和推荐值。

## 五、日常运维
日常巡检、维护操作说明。

## 六、故障处理
常见问题及解决方案。

## 七、附录
参考资料、相关链接等。

---

**修订记录**

| 版本 | 日期 | 作者 | 修改内容 |
|------|------|------|----------|
| v1.0 | YYYY-MM-DD | 作者名 | 初始版本 |
```

### 3. 格式要求

#### 标题层级

- 一级标题：文档主标题（仅一个）
- 二级标题：主要章节
- 三级标题：章节内小节
- 四级标题：具体内容细分

#### 代码块

使用语法高亮的代码块：

````markdown
```bash
# Shell命令示例
systemctl restart nginx
```

```sql
-- SQL语句示例
SELECT * FROM users;
```
````

#### 表格

使用标准Markdown表格格式：

```markdown
| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| port | int | 8080 | 服务端口 |
```

#### 提示信息

使用引用格式标注重要信息：

```markdown
> **注意**：此操作不可逆，请谨慎执行。

> **提示**：建议在测试环境验证后再应用到生产环境。
```

### 4. 命名规范

#### 文件命名

- 使用中文命名，便于识别
- 格式：`组件名称_文档类型.md`
- 示例：`Redis运维手册.md`、`MySQL备份恢复.md`

#### 目录命名

- docs/ 下按技术领域划分
- 使用中文目录名，便于管理

### 5. 提交规范

#### 提交信息格式

```
[类型] 简要描述

类型说明：
- 新增：新增文档
- 更新：更新文档内容
- 修复：修正文档错误
- 删除：删除文档
```

#### 示例

```
[新增] MongoDB运维手册
[更新] Redis运维手册-补充集群配置章节
[修复] Nginx配置说明-修正端口号错误
```

### 6. 修订记录

每个文档底部必须包含修订记录表格，记录每次修改：

```markdown
---

**修订记录**

| 版本 | 日期 | 作者 | 修改内容 |
|------|------|------|----------|
| v1.0 | 2026-01-01 | 张三 | 初始版本 |
| v1.1 | 2026-01-15 | 李四 | 新增故障处理章节 |
```

## 文档审核流程

1. 作者编写/更新文档
2. 提交 Pull Request
3. 团队成员审核
4. 审核通过后合并

## 文档质量要求

- 内容准确、完整
- 步骤清晰、可操作
- 配置参数有说明
- 包含必要的示例
- 及时更新维护
